# Harness Customer User Learning

- [2026-05-20] 고객 테스트는 개발자 의도가 아니라 첫 사용자가 화면에서 이해한 내용을 기준으로 쓴다.
- [2026-05-20] 첫 가치 도달 시간, 첫 클릭, 막힌 단어, 되돌아간 지점을 기록하면 개선 우선순위가 선명해진다.
- [2026-05-20] 개선 제안은 구현 세부가 아니라 사용자 흐름, 문구, 정보 구조 관점으로 작성한다.
- [2026-05-20] 5초 테스트는 설명문을 읽고 평가하지 않는다. 첫 화면만 보고 “무엇을 하는 제품인지”와 “다음 행동이 무엇인지”를 적는다.
- [2026-05-20] 고객 테스트의 실패는 버그만 의미하지 않는다. 문구가 내부 용어라서 멈추거나, 버튼 위치가 예상과 다르거나, 상태 피드백이 늦어도 FAIL일 수 있다.
- [2026-05-20] persona는 한 문장으로 충분하다. 과하게 상세한 persona는 관찰보다 연기를 유도한다.
- [2026-05-20] “있었으면 하는 것”과 “없었으면 하는 것”을 분리하면 후속 작업을 기능 추가와 마찰 제거로 나누기 쉽다.
- [2026-05-20] 추천은 “사용자가 왜 막혔는지”와 연결한다. 근거 없는 취향 제안은 넣지 않는다.
