# HTML 산출물 규칙

Harness에서 사람이 읽고 승인해야 하는 계획, 설계, 최종 보고서는 HTML을 우선한다.

## 기본 구조

- UTF-8
- 단일 HTML 파일
- 외부 네트워크 자원에 의존하지 않는 CSS
- 제목, 요약, 결정 사항, 위험, 검증 결과를 명확히 구분

## 금지

- 실행하지 않은 검증을 성공처럼 표현하지 않는다.
- 본문이 비어 있는 placeholder HTML을 만들지 않는다.
- 자동으로 브라우저를 열지 않는다. 경로만 보고한다.

## 권장 섹션

- Summary
- Scope
- Decisions
- Evidence
- Risks
- Next Steps

## UX 기준

- 첫 화면에 제목, 현재 결론, 다음 결정이 보여야 한다.
- 긴 문서는 상단 요약과 세부 섹션을 분리한다.
- PASS/FAIL/BLOCKED/UNKNOWN은 색상과 텍스트를 함께 사용한다.
- 표는 모바일에서 가로 스크롤되거나 카드형으로 무너지지 않게 한다.
- 증거 링크는 상대 경로보다 `.harness` 기준 경로를 명시한다.
- 읽는 사람이 “무엇이 끝났고, 무엇이 남았고, 무엇을 결정해야 하는지”를 30초 안에 알 수 있어야 한다.

## 최소 CSS

- 본문 최대폭을 제한한다.
- 코드와 경로는 monospace로 구분한다.
- 위험/결정/검증 결과는 시각적으로 분리한다.
- 다크 모드가 없어도 읽을 수 있는 대비를 유지한다.
