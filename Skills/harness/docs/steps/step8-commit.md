# Step 8: 완료 전 정리

## 목표

요청된 경우 변경 내용을 커밋/푸시 가능한 상태로 정리한다.

## 절차

1. `git status --short`로 변경 파일을 확인한다.
2. 사용자 변경과 Harness 작업 변경을 구분한다.
3. 요청된 경우에만 commit 또는 push를 수행한다.
4. commit/push를 하지 않았다면 이유를 progress에 남긴다.

## 완료 조건

- 변경 파일과 검증 결과가 정리되어 있다.
- destructive git 명령을 사용하지 않았다.
- 최종 보고에 넣을 요약이 준비되어 있다.
