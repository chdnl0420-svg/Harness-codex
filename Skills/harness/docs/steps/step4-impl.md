# Step 4: 구현

## 목표

Step 3 계획에 맞춰 코드를 수정하고, 변경 내용을 검증 가능한 상태로 만든다.

## 절차

1. 변경 전 관련 파일을 읽는다.
2. 기존 코드 스타일과 구조를 따른다.
3. 요청 범위 밖 리팩터링을 하지 않는다.
4. 필요한 테스트 또는 검증 보조 코드를 추가한다.
5. 가능한 검증 명령을 실행한다.
6. 결과를 progress에 기록한다.

## 구현 기준

- 수정한 모든 줄은 사용자 요청 또는 검증 필요성과 연결되어야 한다.
- 실패한 테스트는 숨기지 않는다.
- 외부 파일이나 사용자 변경을 되돌리지 않는다.

## 완료 조건

- 변경 파일 목록이 기록되어 있다.
- 실행한 검증과 결과가 기록되어 있다.
- 남은 위험이나 미검증 영역이 명시되어 있다.
