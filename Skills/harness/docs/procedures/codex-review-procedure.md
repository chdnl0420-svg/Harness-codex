# Codex Review Procedure

## 목적

Harness Step 5에서 리뷰를 일관되게 수행하기 위한 절차다.

## 입력

- 작업 slug
- 현재 diff 또는 명시된 파일 목록
- 구현 계획과 테스트 결과

## 절차

1. `git status --short`와 `git diff --stat`를 확인한다.
2. 사용자가 만든 관련 없는 변경은 리뷰 범위에서 분리한다.
3. 변경된 동작과 테스트를 대조한다.
4. 결함, 회귀, 보안/개인정보, 성능, 테스트 공백을 찾는다.
5. `.harness/reviews/review-<slug>.md`에 결과를 누적한다.

## 도구 fallback

1. 기본은 현재 Codex 세션의 직접 리뷰다.
2. 사용자가 명시적으로 병렬 리뷰를 요청한 경우에만 subagent나 review-swarm 계열 도구를 사용한다.
3. 외부 리뷰 도구가 없으면 리뷰를 멈추지 말고 self-review로 진행한다.
4. 이때 리뷰 파일에 `external_review: unavailable`과 미실행 사유를 남긴다.
5. 실행 가능한 테스트나 정적 분석이 있으면 리뷰 보조 증거로 실행한다.

## 출력 기준

- 발견 사항이 없으면 “막는 이슈 없음”이라고 명시한다.
- 발견 사항은 심각도 순으로 쓴다.
- 추측은 추측으로 표시한다.
- 실행하지 않은 테스트는 “미실행”으로 표시한다.
