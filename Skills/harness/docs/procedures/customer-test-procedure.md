# Customer Test Procedure

## 목적

Harness Step 7에서 최종 사용자 관점의 마찰을 찾는다.

## 입력

- 작업 slug
- 실행 방법 또는 접근 URL
- 핵심 사용자 과업
- QA 결과

## 절차

1. 실제 사용자가 접하는 방식으로 제품을 연다.
2. 첫 화면의 목적과 다음 행동이 보이는지 평가한다.
3. 핵심 과업을 수행하고 클릭 경로를 기록한다.
4. 막힘, 헷갈리는 용어, 불필요한 단계를 기록한다.
5. `.harness/results/customer-<slug>.md`에 결과를 쓴다.

## 출력 기준

- 개발자 내부 용어보다 사용자 언어로 쓴다.
- 개선 제안은 관찰 근거와 함께 쓴다.
- 실행하지 못했으면 `BLOCKED`로 기록한다.
