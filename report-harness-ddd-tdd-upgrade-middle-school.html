한 줄 결론

Harness는 이제 도메인 계약 → 구현 계획 → 실패 테스트 → 구현 → QA 증거 순서로 더 엄격하게 움직이도록 바뀌었습니다.

쉽게 말하면, “일단 만들고 나중에 확인하자”가 아니라 “먼저 약속을 적고, 그 약속을 깨는 테스트를 본 뒤, 코드를 고친다”는 방식입니다.

DDD와 TDD를 쉽게 말하면

DDD: 업무 약속을 먼저 정하기

DDD는 Domain-Driven Design의 줄임말입니다. 한국어로는 도메인 주도 설계입니다. 여기서 도메인은 “이 제품이 다루는 업무 세계”입니다.

예를 들어 세션 실행 기능이라면, 세션, 백엔드, 권한, 저장 상태 같은 말을 먼저 정확히 정합니다. 그래야 코드가 엉뚱한 곳을 고치지 않습니다.

TDD: 테스트를 먼저 쓰기

TDD는 Test-Driven Development의 줄임말입니다. 한국어로는 테스트 주도 개발입니다. 먼저 실패하는 테스트를 만들고, 그 테스트를 통과하도록 코드를 고칩니다.

이 방식은 “진짜 요구사항이 동작했는지”를 확인하게 해 줍니다. 단순히 코드가 예뻐졌는지는 기준이 아닙니다.

새 Harness 흐름

앞으로 Harness는 아래 흐름을 기준으로 움직입니다.

1. 도메인 계약 무엇을 지켜야 하는지 먼저 적습니다.
2. 구현 경계 고칠 파일과 고치면 안 되는 파일을 나눕니다.
3. 실패 테스트 아직 통과하면 안 되는 테스트를 먼저 만듭니다.
4. 구현 테스트가 통과할 만큼만 코드를 고칩니다.
5. QA 증거 스크린샷, 클릭 기록, 저장 상태 같은 증거로 확인합니다.

무엇을 바꿨나

1. 새 공통 문서 추가

skills/harness/docs/ddd-tdd-gates.md를 추가했습니다. 이 문서는 DDD/TDD를 Harness 전체에서 같은 방식으로 적용하는 기준입니다.

2. Step2 강화

도메인 설계에 public contract, invariant, command/event, Given/When/Then 예시를 넣도록 했습니다.

3. Step3 강화

구현 계획에 Contract Traceability Matrix와 Test Design Matrix를 넣도록 했습니다.

4. Step4 강화

실패 테스트가 도메인 계약과 연결되어야 합니다. 계약을 일부러 깨는 테스트도 필요합니다.

5. Step6 강화

QA가 도메인 계약을 실제로 검증했는지 확인합니다. 빠진 계약이 있으면 통과 처리하지 않습니다.

6. 템플릿 강화

red test, plan, architecture 템플릿에 계약 추적용 항목을 추가했습니다.

이제 막는 실수

예전에는 생길 수 있던 문제 이제 막는 방법
요구사항을 대충 이해하고 바로 코드를 고침 먼저 Domain Contract를 적어야 함
고치면 안 되는 파일까지 건드림 허용 파일과 금지 파일을 matrix에 기록해야 함
구현 후에 테스트를 끼워 맞춤 Step4 전에 red test가 없으면 TDD_MISSING
정상 케이스만 보고 통과 처리 계약 위반 케이스도 테스트해야 함
QA가 일부 계약을 빼먹음 빠진 contract_id가 있으면 GUIDE_MISSING

딥리서치에서 확인한 근거

리서치는 DDD, TDD, 테스트 피라미드, 계약 테스트, LLM 코드 생성의 약점까지 확인했습니다. 결론은 하나입니다. AI가 코드를 고칠수록, 요구사항과 테스트를 더 분명히 연결해야 합니다.

  • Martin Fowler는 bounded context를 DDD의 핵심 경계로 설명합니다. 출처: Bounded Context
  • Microsoft는 업무 능력과 도메인 경계를 먼저 분석하라고 설명합니다. 출처: Domain Analysis
  • Fowler는 TDD가 실패 테스트, 통과 구현, 정리 순서로 움직인다고 설명합니다. 출처: Test Driven Development
  • 테스트 피라미드는 작은 테스트를 많이, 큰 테스트를 적게 두라고 설명합니다. 출처: Practical Test Pyramid
  • Pact 문서는 계약 테스트를 실행 가능한 예시로 다룹니다. 출처: Pact Docs
  • ContractEval 연구는 AI 코드가 일반 테스트는 통과해도 계약은 어길 수 있음을 보여줍니다. 출처: ContractEval
